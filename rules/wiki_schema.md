# Wiki Schema Rules

You are operating on an LLM Wiki — a **compiled knowledge layer** that
accumulates across many ingest cycles. This file describes the on-disk
contract every wiki page must satisfy. Any output that violates these
rules is rejected by `wiki_compile.py` before reaching the vault.

## Two contours

The vault has two strict contours:

1. **Atomic notes / MOCs / contacts** — generated by `process.py`,
   `process_contacts.py`, `process_notebook.py`. These live under
   `notes_folder`, `moc_folder`, `source_folder`, `contacts_folder`.
2. **LLM Wiki** — every wiki-space lives under
   `<vault>/<wiki_folder>/<project-slug>/`. Wiki pages **never** appear
   outside this root and atomic notes **never** appear inside it.

You must not propose pages that cross this boundary. Do not reference
atomic notes from wiki pages by `[[wikilink]]` — wiki pages link only
to other wiki pages (or to the explicit `[[?slug]]` open-question marker).

## File layout

```
<wiki-space>/
├── SCHEMA.md          (frozen contract, do not modify after init)
├── index.md           (regenerated every compile)
├── log.md             (append-only journal)
├── raw/<kind>/        (immutable inputs; kinds: articles, docs, transcripts, assets)
├── pages/             (core pages — required in project mode only)
├── entities/          (services, integrations, external systems)
├── concepts/          (domain concepts)
├── comparisons/       (A vs B writeups)
└── queries/           (saved questions and their compiled answers)
```

## Machine-owned pages (do not hand-edit)

Compiled pages — everything under `pages/`, `entities/`, `concepts/`,
`comparisons/`, `queries/`, `readouts/` — are **machine-owned**.
`wiki_compile.py` rewrites them from `raw/` + the existing snapshot via the
LLM on every compile. Hand-edits to their prose are **not guaranteed to
survive** the next compile; only `[[wikilinks]]` are protected (the
`WIKI_LINKS_LOST` safety property). To add or correct durable knowledge,
add a `raw/` input and recompile — do not edit compiled pages by hand.

(`SCHEMA.md` is the exception: it is frozen and never touched by compile.)

## Page types

`wiki_page_type` is one of:

| Value | Where it lives | When to use |
|-------|---------------|-------------|
| `core` | `pages/` | Fixed scaffold pages in project mode |
| `entity` | `entities/` | A discrete system, service, integration, person |
| `concept` | `concepts/` | A reusable domain idea or pattern |
| `comparison` | `comparisons/` | An explicit A-vs-B analysis |
| `query` | `queries/` | A useful question with its compiled answer |
| `raw` | `raw/<kind>/` | Original input (created by `wiki_ingest.py`) |
| `meta` | `<wiki-space>/` (root) | SCHEMA / index / log only |

## Core pages (project mode)

A project-mode wiki-space must have all seven core pages under `pages/`:

- `overview.md` — what this project is, in 3-5 sentences
- `architecture.md` — high-level shape, layers, data flow
- `components.md` — major components with links to entities
- `workflows.md` — end-to-end flows the project supports
- `goals-and-roadmap.md` — explicit goals (cited from raw)
- `glossary.md` — short-form definitions; deeper ones promote to `concepts/`
- `open-questions.md` — unresolved questions and contradictions

Corpus mode does not require these — only entities, concepts, etc. grow as
raw is added.

## Frontmatter contract

Required on every page:

- `note_type: wiki`
- `wiki_project: <slug>` (must match enclosing folder)
- `wiki_page_type`: see table above
- `wiki_status`: one of `stub | draft | stable | stale | contradicted | ingested`
- `date`: ISO date of last touch
- `source_doc`: synthetic id `wiki:<slug>:<page_type>:<stem>`
  (raw notes may carry the original filename instead)

Optional:

- `confidence`: `high | medium | low` — pages tagged `low` must open with a
  `> [!warning]` callout that names the gap
- `sources`: list of `raw/...md` paths the page draws on
- `related`: list of slugs (NOT `[[wikilinks]]`) for cross-reference
- `last_compiled`: ISO timestamp of last compile that touched the page

## Wikilink rules

- Targets must resolve to another wiki page **inside the same wiki-space**
  (matched by stem or by H1 title).
- Use `[[?slug]]` to mark a link to a page that does not yet exist but
  should — `wiki_compile.py` records these as open questions.
- Never link from a wiki page to an atomic note or to an external file.

## Slug format

Slugs (project, page stems, raw filenames) are kebab-case ASCII:
`^[a-z0-9][a-z0-9-]*$`. No spaces, no underscores, no uppercase, no
unicode.

## Status lifecycle

`stub → draft → stable` is the normal lifecycle. `stale` flags pages whose
sources have been superseded but not yet rewritten. `contradicted` flags
pages that conflict with another page in the same wiki-space (pair must
also appear in `pages/open-questions.md`). `ingested` is reserved for raw
notes after compile has consumed them.

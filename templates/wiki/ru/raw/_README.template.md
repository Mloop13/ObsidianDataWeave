# raw/ — {{title}}

Эта папка хранит **неизменяемые входы**, из которых читает каждый проход
компиляции. Файлы здесь никогда не модифицируются wiki-скриптами — они
только добавляются `wiki_ingest.py` и потребляются (только на чтение)
`wiki_compile.py`.

## Подпапки

- `articles/` — длинные тексты, статьи, посты в блогах, научные работы
- `docs/` — справочные доки, README-файлы, дизайн-доки
- `transcripts/` — стенограммы встреч, подкастов, интервью
- `assets/` — фрагменты кода, JSON-дампы, скриншоты, всё остальное

## Соглашение об именах

`<YYYY-MM-DD>-<slug>.md`. Slug'и — kebab-case ASCII; `wiki_ingest.py`
генерирует их автоматически из имени входного файла.

## Зачем отдельный слой raw

Слой raw — это **источник истины**. Скомпилированные wiki-страницы можно
перегенерировать из raw + SCHEMA в любой момент. Страницы в `entities/`,
`concepts/` и т.д. — это *интерпретации*: они накапливают, обобщают и
связывают, но именно raw остаётся аудит-следом.
